# Contributing to EZ Pass FL


Thank you for your interest in contributing to **EZ Pass FL**! We welcome contributions of all kinds â€” code, documentation, design, bug reports, and feature suggestions.


## Contribution Guidelines


- Be respectful and professional. Follow this project's Code of Conduct.
- Keep pull requests small and focused on a single issue/feature.
- Write clear commit messages and include a brief description of what your change does.


## How to Contribute


1. Fork the repository.
2. Create a branch for your changes: `git checkout -b feature/your-feature-name`.
3. Make your changes on the branch and commit often.
4. Push to your fork: `git push origin feature/your-feature-name`.
5. Open a pull request to the `main` branch of the upstream repository.


## Reporting Issues


If you find a bug or problem:


- Check existing issues to avoid duplicates.
- Create a new issue with a descriptive title and steps to reproduce, expected vs actual behavior, and any relevant screenshots or logs.


## Suggesting Features


Want to propose a new feature?


- Open an issue with the tag `enhancement`.
- Describe the use case, who benefits, and a rough sketch of how it should behave.


## Code of Conduct


By participating you agree to:


- Be kind and inclusive.
- Refrain from abusive or discriminatory language.
- Give respectful feedback and be open to receiving it.


If you experience or witness harassment, please report it to the maintainers.


## Review Process


- Maintainers will review pull requests and aim to respond within a few business days.
- Reviews focus on correctness, readability, tests, and alignment with project style.
- Contributors may be asked to make changes; please address review comments promptly.
- After approval, a maintainer will merge the PR.


Thank you for helping make EZ Pass FL better!
